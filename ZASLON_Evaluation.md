# Оценка проекта ZASLON (Утилита аварийного восстановления)

## 1. Общий архитектурный обзор
ZASLON — это мощная утилита уровня пользователя (User Mode) для ОС Windows, написанная на C++ с использованием графического API Direct3D 9 и фреймворка ImGui. Проект спроектирован как агрессивный антивирусный сканер и инструмент для экстренного восстановления системы (в духе AVZ или утилит от Kaspersky/Dr.Web, но с упором на независимость и работу в WinPE).

**Ключевые особенности архитектуры:**
*   **Минимум зависимостей:** Проект компилируется со статической линковкой рантайма (`/MT`), что делает бинарник переносимым и позволяет запускать его в средах с ограниченными ресурсами, таких как Windows PE (Preinstallation Environment).
*   **Инкапсуляция GUI и Логики:** В проекте четко разделены модули интерфейса (ImGui панели: `dashboard.cpp`, `process_panel.cpp` и т.д.) и низкоуровневой системной логики (`zaslon_core.cpp`, `offline_os_manager.cpp`, `system_repair.cpp`).
*   **Грамотный Fallback-механизм отрисовки:** При инициализации D3D9 (`main.cpp`) последовательно запрашивается аппаратное, программное (для WinPE/базовых видеодрайверов) и референсное ускорение. Также вкомпилированы шрифты (через байт-массивы), чтобы интерфейс не превратился в «квадраты», если системные шрифты повреждены вирусом.

## 2. Анализ нововведений версии 2.7.0 («Оживление»)
На основе анализа справки (`help_tab.cpp`) и исходного кода (`offline_os_manager.cpp`, `system_repair.cpp`), версия 2.7.0 является масштабным обновлением, сфокусированным на спасении «мертвых» систем через WinPE.

**Ключевые фичи v2.7.0:**
*   **Smart WinPE (Умный поиск ОС):** Модуль `OfflineOSManager` автоматически сканирует логические диски (игнорируя RAM-диск `X:` среды WinPE) в поисках папки `\Windows\System32\config\SYSTEM`.
*   **Offline Registry (Умный реестр):** Реализован элегантный механизм монтирования кустов реестра сломанной ОС (SOFTWARE и SYSTEM) во временные разделы `HKEY_LOCAL_MACHINE\ZASLON_OFF_SOFTWARE` с автоматической прозрачной трансляцией путей (функция `OpenRedirectedKey` и `RedirectOfflinePath`).
*   **Тяжелая артиллерия (DISM & SFC Offline):** ZASLON научился формировать правильные команды для `sfc.exe /offbootdir=... /offwindir=...` и `dism.exe /image:...`, позволяя восстанавливать системные файлы мертвой ОС прямо с флешки.
*   **Ремонт загрузчика:** Добавлена кнопка, которая в фоне асинхронно пробрасывает команды `bootrec /fixmbr`, `bootrec /fixboot` и `bcdboot`, восстанавливая MBR, Boot и BCD.
*   **Безопасная очистка:** Поскольку ZASLON работает из-под WinPE (где вирусы не загружены в память), функция очистки автозагрузки, планировщика задач и WMI-подписок работает безотказно (вирусы спят и не могут защищаться).

## 3. Оценка реализации системных модулей (ZaslonCore)

Реализация функционала в `zaslon_core.cpp` выполнена на очень высоком экспертном уровне использования Win32 API и Native API:

### 3.1. The Terminator (Убийца процессов)
Функция `ForceKillProcess` содержит **7 уровней** устранения процесса, что является выдающимся решением для борьбы с защищающимися руткитами/локерами:
1.  Стандартный `TerminateProcess`.
2.  Прямой вызов Native API: `NtTerminateProcess` (помогает при User-Mode хуках).
3.  Подключение отладчика (`DebugActiveProcess`) и мгновенное его отключение с завершением.
4.  Инъекция: `CreateRemoteThread` с вызовом `ExitProcess` внутри адресного пространства вируса.
5.  Объект-Задание (Job Object): помещение защищенного процесса в Job с последующим `TerminateJobObject`.
6.  Снайперское уничтожение всех потоков через `Thread32First`/`OpenThread`/`TerminateThread`.
7.  **Handle Closing Attack (Resource Starvation):** Гениальный трюк с использованием `NtQuerySystemInformation` для перечисления всех дескрипторов (Handles) целевого процесса и их принудительного закрытия через `DuplicateHandle(..., DUPLICATE_CLOSE_SOURCE)`, что заставляет процесс-жертву рухнуть из-за нехватки ресурсов.

### 3.2. Heuristic Risk Engine (Движок эвристики)
Модуль оценки рисков процессов (0-100%) написан очень солидно:
*   Считывает цифровые подписи (`WinVerifyTrust`).
*   Проверяет пути запуска (Temp, AppData, Downloads).
*   Анализирует **PE-заголовки**: находит .NET-сигнатуры, проверяет секции на предмет упаковщиков (UPX, VMP, Themida).
*   Вычисляет **Энтропию Шеннона** для определения запакованных/зашифрованных секций.
*   Сканирует **Таблицу Импортов (IAT)** на наличие подозрительных API (например, `WriteProcessMemory`, `CreateRemoteThread`), что характерно для инжекторов.
*   Проверяет `Zone.Identifier` (поток Alternate Data Stream, указывающий на скачивание файла из сети).
*   Сравнивает разрядности, имена процессов и их родителей (например, детектирует фальшивый `svchost.exe`).

### 3.3. Smart Unlocker
Использует API Restart Manager (`Rstrtmgr.dll`) для определения процессов, заблокировавших файл. Сначала программа просит их корректно завершиться (`RmShutdown`), а если это не помогает — безжалостно убивает через `ForceKillProcess`.

## 4. Качество кода и вердикт

**Сильные стороны:**
*   **Глубокое понимание Windows Internals:** Код демонстрирует отличное знание недокументированных функций Windows (Native API, структуры PE32/PE32+).
*   **Надежность:** Широкое использование асинхронности (`std::thread`, `std::atomic`), чтобы ресурсоемкие задачи (DISM, SFC) не вешали ImGui-интерфейс. Каждая критическая операция завернута в проверки ошибок.
*   **Безопасность:** Регистрация глобальных хоткеев (Ctrl+Alt+Z — Anti-WinLocker) для вызова изолированного рабочего стола (`CreateDesktopW`) — отличное средство защиты от окон-вымогателей, перекрывающих экран.

**Потенциальные зоны для улучшения:**
*   **Отсутствие драйвера (Kernel Mode):** Несмотря на агрессивность 7-уровневого терминале, некоторые современные Rootkit-угрозы перехватывают вызовы на уровне Ring0 (SSDT хуки/ObCallbacks). Без `.sys` драйвера у User-Mode утилиты есть предел возможностей на *живой* зараженной системе (хотя WinPE-модуль из v2.7 изящно решает эту проблему).
*   **Стиль кода:** В некоторых местах встречаются спартанские решения в стиле Си внутри C++ проекта (использование `malloc`/`free` для получения токенов вместо смарт-поинтеров и `std::vector`), что может теоретически вести к утечкам памяти в случае C++ исключений, хотя использование `__try` / `__except` отчасти это компенсирует.

### Итог
**ZASLON v2.7.0** — это профессионально написанный, узкоспециализированный инструмент, блестяще справляющийся со своей задачей. Переход к концепции «Умного WinPE» в версии 2.7.0 сделал программу ультимативным инструментом сисадмина, так как оффлайн-лечение (вне зараженной среды) лишает вирусы 100% их защитных механизмов. Архитектура продумана, модульна и готова к расширениям. Общая оценка реализации проекта: **Отлично (9.5/10)**.

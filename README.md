# ZASLON — Утилита аварийного восстановления Windows

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![C++](https://img.shields.io/badge/C%2B%2B-17-blue.svg)](https://isocpp.org/)
[![Platform](https://img.shields.io/badge/platform-Windows%2010%2B-lightgrey.svg)](https://www.microsoft.com/windows)
[![Build](https://img.shields.io/badge/build-CMake%203.25+-green.svg)](https://cmake.org/)

**ZASLON** — это профессиональная утилита для экстренного восстановления и лечения заражённых систем Windows. Инструмент разработан для работы в средах с ограниченными ресурсами, включая **Windows PE (WinPE)**, и обеспечивает оффлайн-доступ к реестру, файловой системе и компонентам повреждённой ОС.

> 💡 **Идеально подходит для:** системных администраторов, специалистов по информационной безопасности, энтузиастов восстановления данных.

---

## 🚀 Особенности

### 🔥 Ключевые возможности

| Модуль | Описание |
|--------|----------|
| **The Terminator** | 7-уровневая система принудительного завершения процессов (включая Native API, инъекции, Job Objects) |
| **Heuristic Risk Engine** | Эвристический анализ процессов (0–100%) с проверкой подписей, PE-заголовков, энтропии и импортов |
| **Smart WinPE** | Автоматический поиск и монтирование оффлайн-систем из среды WinPE |
| **Offline Registry** | Прозрачная работа с реестром повреждённой ОС через монтирование кустов |
| **DISM & SFC Offline** | Восстановление системных файлов через встроенные инструменты Windows |
| **Bootloader Repair** | Автоматическое восстановление MBR, Boot и BCD загрузчиков |
| **Smart Unlocker** | Разблокировка файлов через Restart Manager API |
| **Anti-WinLocker** | Горячие клавиши (Ctrl+Alt+Z) для вызова изолированного рабочего стола |

### 🛡️ Преимущества

- **Минимум зависимостей** — статическая линковка CRT (`/MT`), переносимый бинарник
- **Работа в WinPE** — вирусы не активны, что позволяет лечить систему «на холодную»
- **DirectX 9 + ImGui** — современный графический интерфейс с поддержкой тем
- **Асинхронность** — тяжёлые операции не блокируют UI
- **Встроенные DLL** — критические библиотеки извлекаются из ресурсов

## 🏗️ Сборка проекта

### Предварительные требования

1. **Visual Studio 2022** с компонентом `Microsoft.VisualStudio.Component.VC.Tools.x86.x64`
2. **CMake 3.25+** (добавьте в `PATH`)
3. **PowerShell** (для подготовки ресурсов)

### Быстрая сборка

```batch
:: Сборка Release-версии (x64)
build.bat

:: Сборка Debug-версии
build.bat x64 Debug

:: Очистка артефактов сборки
build.bat clean
```

### Ручная сборка через CMake

```bash
# Конфигурация
cmake --preset x64-Release

# Сборка
cmake --build --preset x64-Release --parallel

# Результат: bin/Release/zaslon.exe
```

### Зависимости

Проект автоматически загружает следующие зависимости через `FetchContent`:

- **[ImGui](https://github.com/ocornut/imgui)** (ветка `docking`) — графический интерфейс
- **[stb_image](https://github.com/nothings/stb)** — загрузка изображений

Никаких дополнительных действий не требуется — CMake всё настроит автоматически.

## 📖 Использование

### Запуск в обычной системе

Просто запустите `zaslon.exe` от имени администратора:

```batch
zaslon.exe
```

### Запуск в WinPE (рекомендуется)

1. Создайте загрузочную флешку с WinPE (например, через [Sergei Strelec](https://sergeistrelec.ru/) или [MediCat](https://medicatusb.com/))
2. Скопируйте `zaslon.exe` на флешку
3. Загрузитесь с флешки в WinPE
4. Запустите утилиту — она автоматически найдёт установленные системы Windows

> ⚠️ **Важно:** Для полноценной работы требуется запуск от имени **Администратора**. В WinPE это обычно выполнено по умолчанию.

### Горячие клавиши

| Комбинация | Действие |
|------------|----------|
| **Ctrl+Alt+Z** | Вызов изолированного рабочего стола (Anti-WinLocker) |
| **F1** | Открыть вкладку помощи |

## 🔬 Технические детали

### 7 уровней уничтожения процессов

1. `TerminateProcess` — стандартный API
2. `NtTerminateProcess` — Native API (обход пользовательских хуков)
3. Отладчик + мгновенный выход
4. Инъекция `ExitProcess` через `CreateRemoteThread`
5. Job Object + `TerminateJobObject`
6. Уничтожение всех потоков через `TerminateThread`
7. **Handle Closing Attack** — закрытие всех дескрипторов процесса (Resource Starvation)

### Эвристический анализ рисков

Движок оценивает процессы по шкале 0–100% на основе:

- ✅ Цифровой подписи (`WinVerifyTrust`)
- ✅ Пути запуска (Temp, AppData, Downloads = подозрительно)
- ✅ PE-заголовков (секции, упаковщики UPX/VMP/Themida)
- ✅ Энтропии Шеннона (запакованные/зашифрованные секции)
- ✅ Таблицы импортов (подозрительные API: `WriteProcessMemory`, `CreateRemoteThread`)
- ✅ Потока `Zone.Identifier` (файлы из интернета)
- ✅ Соответствия имени процесса и пути (детект подделок типа `svchost.exe`)
